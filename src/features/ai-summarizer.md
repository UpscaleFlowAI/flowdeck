# Feature Spec: AI Workflow Summarizer

## Problem

Automation agency operators spend significant time each week manually reviewing workflow execution logs to answer basic questions: Which workflows failed most often? Which client's automations consumed the most execution time? Are there patterns in failures that point to a systemic issue? These questions are simple to ask but tedious to answer because the data is spread across multiple platforms and hundreds of individual execution records. Most operators either skip this review entirely or do it superficially once a month, which means problems compound silently until a client escalates.

## Solution

The AI Workflow Summarizer adds a natural-language query interface to the FlowDeck dashboard. Operators can type a plain-English question — such as "which Zaps failed most this week?" or "show me Make scenarios that took longer than 30 seconds on average" — and receive a one-paragraph summary generated by analyzing the stored execution metadata. The summarizer works across all connected platforms, so a question like "what broke the most across all clients?" returns a unified answer, not three separate platform-specific responses. Queries are scoped to the user's connected workspaces and respect workspace grouping, so an operator can ask about a specific client's workflows or the entire portfolio.

## User Impact

This feature turns FlowDeck from a monitoring tool into an intelligence layer. Instead of scanning rows in a table and mentally aggregating patterns, operators get direct answers to the questions they actually care about. A weekly client review that used to require 20 minutes of log-diving can now start with a single question: "what should I know about this client's workflows this week?" The summarizer surfaces patterns that would be easy to miss in a table view — like a workflow that passes 95% of the time but consistently fails on Mondays, suggesting a dependency on a service that has Monday maintenance windows. For agencies managing 10+ clients, this is the difference between reactive firefighting and proactive service.

## Release Notes (one-liner)

Adds an AI-powered natural-language query interface that lets operators ask questions about their workflow history and get instant one-paragraph summaries across all connected platforms.

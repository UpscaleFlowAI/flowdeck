# Marketing Brief — FlowDeck

## Brand Voice

**Confident, practical, slightly playful.** We talk like a sharp colleague who's been in the automation trenches, not like a SaaS landing page. No corporate jargon, no "leverage synergies," no "empower your workflow ecosystem." Say what the product does, say why it matters, and stop.

- **Do:** "FlowDeck shows you what's broken before your client notices."
- **Don't:** "FlowDeck empowers agencies to leverage unified observability across heterogeneous automation ecosystems."

## Target Audience

- **Primary:** Automation agency owners and operators — people who manage n8n, Make, and Zapier workflows for multiple clients as a business
- **Secondary:** Solo automation builders who freelance or consult on workflow design
- **Geography focus:** MENA region (Lebanon, UAE, Saudi Arabia, Egypt, Jordan) + global English-speaking automation community

## Tone for Arabic Content

When writing in Arabic, use **Lebanese dialect** (not MSA / فصحى). The tone should be:

- **Warm and direct** — like you're talking to a friend who runs an automation agency
- **Practical** — focus on what the feature does for them, not abstract value props
- **Confident but not arrogant** — we know this tool is useful, we don't need to oversell it

## Transliteration Rules

When writing Arabic content, **transliterate all English product names and technical terms to Arabic script**:

| English | Arabic |
|---------|--------|
| FlowDeck | فلو ديك |
| GitHub | جيت هاب |
| Claude | كلود |
| n8n | إن ايت إن |
| Make | مايك |
| Zapier | زابيير |
| API | أي بي آي |
| Slack | سلاك |
| OAuth | أو أوث |
| dashboard | داشبورد |
| webhook | ويب هوك |

**Exceptions — keep in English:**
- Shell commands and code snippets
- URLs and links
- Version numbers (v0.4.0)
